# aeon v0.1.0 is live — and the agent repo ate its own cooking

**Window: 2026-07-09 → 2026-07-13**

By-the-numbers: **36 PRs merged** (23 aeon · 11 miroshark-aeon · 1 MiroShark · 1 miroshark-website) across 4 days. ~29 commits to aeon, 1 to MiroShark. Stars: MiroShark 1,360 (+3), aeon 574 (+3).

---

## What shipped

### aeon v0.1.0 — first stable release

The headline this window. After months of iteration across 70 partners, the first stable release landed on Jul 10.

What went into it (condensed from the release thread on x.com/aeonframework):
- Grok + multi-harness support
- Telegram instant mode, new commands, email channel
- Global codebase simplification — fewer skills, fewer dependencies
- Open Knowledge Format (OKF) support
- Attestation support for skills — prove what you ran, onchain or offchain
- Langfuse observability opt-in

The architectural shift underneath: **retire postprocess, run everything in-run** (PRs #695, #707). Irreversible sends — deploys, token transfers, emails — now fire via `./secretcurl` as the skill's final action, not in a detached postprocess step. One failure surface, not two.

The README went from 733 → 596 lines (PRs #700–#712), got a CHANGELOG.md seeded at v0.1.0 (#712), a "Proof of Work" section, and cleaner Authentication + Notifications setup. Community packs got their own listing — first entry: AgentLink (PR #686, `techdigger/aeon-skill-pack-agentlink`).

Other quality fixes that landed alongside: model backstop aligned (opus-4-8 → sonnet-4-6, #698), transient skill writes routed to /tmp not repo root (#713), empty_output scorer fixed to only flag empty FINAL outputs not legitimate no-ops (#714), swallowed errors in MCP CLI surfaced (#716), color tokens unified eva-*/aeon-* → aeon-* (#717).

### miroshark-aeon migrates onto v0.1.0 template (PR #102)

The agent repo itself adopted the new stable template. State preserved. Non-template files cleaned out (.mcp.json, secretcurl, .chains, ECOSYSTEM/SHOWCASE). Runtime artifacts rooted in /tmp. The scorer fix (#112) and token-movers routing fix (#108) were pulled in parallel.

Eating its own cooking is the point. If the framework is clean enough to migrate the agent that runs it, it's ready for partners.

### MiroShark — quiet week

One Dependabot bump (#244, soupsieve in /backend). The new website section went live (x.com/miroshark_/status/2076033274862841863) — not a code ship but signal the front-facing story is moving.
