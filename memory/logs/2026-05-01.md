### Endpoints added
- `GET /api/simulation/<id>/trajectory.csv` — RFC 4180 CSV, locked column order: `round, round_timestamp, bullish_pct, neutral_pct, bearish_pct, participating_agents, total_posts, total_engagements, quality_health, participation_rate`. Header row emitted even on empty trajectories so consumers don't have to special-case zero-row files. `Content-Disposition: attachment` so a click triggers save-as without an explicit `download` attribute.
- `GET /api/simulation/<id>/trajectory.jsonl` — newline-delimited JSON, one object per line, same field shape and order as the CSV row. Empty input yields zero bytes (no header — well-formed JSONL has no header concept). DuckDB `read_json_auto`, Pandas `read_json(lines=True)`, and stream-processing pipelines (Kafka, Beam, Materialize) consume natively.

Both share the existing transcript / share-card publish gate (`is_public=true`), `Cache-Control: public, max-age=60`, and the same ±0.2 stance threshold every other surface uses (gallery / share card / replay GIF / transcript / webhook / feed).

### Files
- **NEW** `backend/app/services/trajectory_export.py` (297 lines) — pure stdlib (`csv` + `io` + `json`). `compute_stance_split()` helper with strict-inequality bucketing at the boundary (exactly ±0.2 ⇒ neutral). `build_rows()` reads `trajectory.json` + `quality.json` and emits one analyst row per snapshot; defensive sort on `round` so out-of-order on-disk snapshots still emit chronologically. `_participating_agents()` counts distinct `viral_posts` user_ids and falls back to `active_agent_count` when a quiet round has no posts. `render_csv` uses `csv.DictWriter` + `QUOTE_MINIMAL` so numeric columns stay unquoted (`pd.read_csv()` dtype inference works without a converters argument). `render_jsonl` filters output to the canonical `CSV_COLUMNS` field list so an upstream change to `_row_from_snapshot` can't accidentally leak unstable keys into the JSONL stream.
