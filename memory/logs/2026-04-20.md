## Agent Self-Improvement — XAI Cache Query Validation

- **Trigger:** Today's log shows `fetch-tweets` ran three times (first run returned 0 new, re-run 1 discovered "Cache query mismatch: prefetch used `$AEON OR @miroshark_ OR github.com/aaronjmars/miroshark` (default var), not `$MIROSHARK`", re-run 2 finally got a correct cache with 11 new tweets). A stale `$AEON`-era cache served empty `output_text` (4 unreachable URL annotations) while `aeon.yml` has been on `$MIROSHARK` for days — the skill had no way to detect it was consuming a wrong-query cache.
- **Fix:**
  - `scripts/prefetch-xai.sh` — fetch-tweets branch now `rm -f`s the existing `.xai-cache/fetch-tweets.json` + `.xai-cache/fetch-tweets.query` before calling `xai_search`, so a failed API call can't leave stale cache content. On success it writes the current `$VAR` verbatim to `.xai-cache/fetch-tweets.query`.
  - `skills/fetch-tweets/SKILL.md` — Path A now reads `.xai-cache/fetch-tweets.query` and only consumes the cache if the sidecar matches the current `${var}`. Mismatch (or missing sidecar) falls through to Path B (live X.AI call) instead of serving wrong-query results.
- **Impact:** Eliminates the "stale-cache silent failure" mode that burned two fetch-tweets invocations today. Any future `aeon.yml` var change (e.g. new token) will now self-heal on the next prefetch instead of requiring operator re-runs.
