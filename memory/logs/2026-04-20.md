## Feature Built — Scenario Auto-Suggest

- **PR:** https://github.com/aaronjmars/MiroShark/pull/39 `feat/scenario-auto-suggest`
- **Source idea:** `articles/repo-actions-2026-04-18.md` idea #1 (Scenario Auto-Suggest from Document — the last major setup friction point before first-value)
- **Dedup check:** open PRs #36 (mbs5 perf), #37 (Counterfactual Explorer), #38 (mbs5 embedding-config fix) — none overlap with scenario auto-suggest
- **What ships:**
  - Backend `POST /api/simulation/suggest-scenarios` — accepts `{ text_preview, no_cache? }`, normalizes + SHA-256s + LRU-caches (cap 64), clamps to 2000 chars, calls `create_llm_client().chat_json()` (20s timeout) with compact Bull/Bear/Neutral system prompt, `_clean_suggestions` validates labels + ranges (0–100, swap if reversed) + question lengths, always 200s on soft failures with `reason` code
  - Frontend `ScenarioSuggestions.vue` — debounced (800ms, min 120 chars) card grid with Bull/Bear/Neutral colored left-borders + badges, loading spinner, dismiss, monotonic `requestSeq` guard against late-response overwrites
  - `Home.vue` — reads `.md`/`.txt` client-side (File.slice(0, 6000).text(), 6KB cap), combines with `urlDocs[].text`, clamps to 6KB, hands to component; `handleSuggestionUse` fills `formData.simulationRequirement`
  - `api/simulation.js` — `suggestScenarios(...)` with 25s override timeout
  - README — new "Smart Setup (Scenario Auto-Suggest)" subsection under How It Works
- **Non-blocking**: LLM-unavailable / timeout / malformed → 200 + `suggestions:[]` + reason code → panel silently hides, form still works
- **Not included (PDFs)**: PDF text extraction happens backend-side during graph build; PDF-only uploads don't trigger auto-suggest. Follow-up would be a `multipart/form-data` variant using the existing `FileParser`.
