## Self-Improve — bankr-prefetch agent-timeout distinction
- **Skill**: self-improve (even day = scheduled)
- **Trigger**: 3 consecutive `TWEET_ALLOCATOR_EMPTY` runs (May 18 / 19 / 20) — handles previously verified (100xDarren May 17, cybercelos May 14+17, btcbabycow May 14+16) suddenly return null. `scripts/prefetch-bankr.sh` polls the Bankr Agent job 8×8s=64s, then extracts the wallet *whether or not* status reached `"completed"`. Max-Mode Agent calls (claude-sonnet-4.6) chain multiple internal tool calls and can run longer than 64s — when they do, the empty/incomplete `result.text` matches no `0x...` pattern and the handle is silently recorded as null. Same downstream effect as "Bankr genuinely doesn't know," so the skill couldn't tell them apart.
- **Diff**:
  - `scripts/prefetch-bankr.sh` (+~25 / -8): poll loop extended from 8 → 14 iterations (~112s window); submit `--max-time` 30 → 45s; new `TIMED_OUT` counter increments when the poll loop exits without `completed`/`failed`; timed-out handles are **not** written to `verified-handles.json` (separated from `null` entries); new `prefetch-status.json` field `timed_out`; new top-level status `"agent-timeout"` when `VERIFIED==0 && TIMED_OUT > NULL_COUNT`.
  - `skills/tweet-allocator/SKILL.md` (+~3 / -2): new branch in step 4 for `"agent-timeout"` status — logs `TWEET_ALLOCATOR_ERROR` with explicit retry/credit hint instead of silently logging `_EMPTY`; status-flag docstring extended.
