## Feature Built — Search-Engine Sitemap (PR #82)
- **Source:** repo-actions 2026-05-12 idea #5 (last unbuilt idea from that batch)
- **Branch:** `feat/sitemap-xml-search-discovery` → MiroShark
- **PR:** https://github.com/aaronjmars/MiroShark/pull/82
- **What it does:** Auto-generates `GET /sitemap.xml` (sitemaps.org 0.9 XML) over the public-simulation corpus — one `<url>` block per `/share/<id>` (priority 0.8) and per `/watch/<id>` (priority 0.7). Companion `GET /robots.txt` always served, advertises sitemap via standard `Sitemap:` directive when enabled, always carries `Disallow: /api/` to keep JSON namespace out of search index. New `GET /api/config/sitemap` exposes the public flag to the SPA. Public-only filter, `is_public=false` sims silently excluded. Capped at sitemap-spec ceiling 50,000 entries.
- **Files modified (14):** backend/app/services/sitemap.py (~334 LoC, new), backend/app/api/sitemap.py (+158, new), backend/tests/test_unit_sitemap.py (+361, new), backend/app/__init__.py (+5), backend/app/api/__init__.py (+4), backend/app/config.py (+10), backend/openapi.yaml (+111), docs/FEATURES.md (+21), docs/API.md (+16), docs/API.zh-CN.md (+16), frontend/src/api/simulation.js (+36), frontend/src/components/EmbedDialog.vue (+70), README.md (+2), .env.example (+13)
- **Notable design calls:** (1) Sims sorted by `simulation_id` ascending so two consecutive renders against the same on-disk corpus produce byte-identical XML — lets the route layer set a meaningful ETag if it ever wants to. (2) `<lastmod>` falls back through `updated_at` → `created_at` → `state.json` mtime so a long-lived in-progress sim whose `created_at` is days old still tells the crawler "the artifact changed today". (3) `<changefreq>always</changefreq>` for in-progress sims (belief bars genuinely change every round); `weekly` for completed share entries / `daily` for completed watch entries. (4) `ENABLE_SITEMAP=false` makes `/sitemap.xml` 404 AND drops the `Sitemap:` line from `robots.txt` — no leak through robots either. (5) `Disallow: /api/` always served regardless of sitemap flag — even private deployments need crawler hygiene.
- **Streak:** 20 consecutive zero-new-deps PRs (#57 → #82). Pure stdlib `xml.etree.ElementTree` + `os` + `datetime`.
- **Tests:** could not run pytest locally (sandbox blocked); 22 new tests cover pinned invariants, public/private filtering, lastmod fallback chain, changefreq semantics, byte determinism, MAX_SITEMAP_URLS cap, robots.txt directives, XML round-trip via `ET.fromstring`, and 5 drift-detection guards (route decorators present, blueprint exported + mounted, Config flag declared).
- **Closes the May-12 batch:** all 5 ideas now resolved — #1 Lifecycle Webhooks (deferred — simulation-runner hook-point identification is higher risk), #2 Interactive Embed Widget (already exists as SPA route `/embed/:simulationId`), #3 Filtered RSS/Atom (PR #81, 2026-05-13), #4 Per-Round Belief Snapshot (already exists as `/frame/<round_num>` since pre-PR #57), #5 Sitemap (this PR). Three of the five were redundant or already shipped — the build cycle is the right unit of waste, but the exploration cost is real. **Lesson:** `feature` skill should grep the existing routes / SPA before committing to a repo-actions idea.
