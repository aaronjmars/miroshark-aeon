- **What it does:** `POST /api/simulation/batch-status` with body `{"sim_ids": [...]}` (1–20 ids) returns one entry per id in input order. Per-id publish gate hides private + unknown ids behind a byte-identical `{found: false, ...nulls}` envelope (a caller cannot probe for private-sim existence). Completed sims emit `direction` / `confidence_pct` / `quality_health` / `total_rounds` / `completed_at` derived byte-for-byte from `signal_service.compute_signal` (same plurality + tie-break as per-sim signal.json). Running / failed / cancelled emit bare `status` + null analytics. Duplicates in input emit duplicate entries (list semantics, not set).
