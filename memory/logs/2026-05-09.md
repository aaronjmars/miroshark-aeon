## Feature — Simulation Lineage Navigator (PR #76 on MiroShark)

- Built and shipped May 8 repo-actions idea #3 (Simulation Lineage Navigator) — direct follow-up to PR #75 (Reproducibility Config Export). Picked over #1 Trending Sort, #2 oEmbed Endpoint, #4 Peak-Round Snapshot, #5 Operator Profile because it surfaces metadata that's already on disk from yesterday's PR #75 and turns a one-directional pointer into a navigable graph
- Closes the **lineage-navigation gap** PR #75 created — `parent_simulation_id` + `lineage.kind` + counterfactual trigger metadata persisted in every reproduce.json was one-directional (child knew its parent, parent had no view into its children). A researcher running 3 counterfactual branches off a base scenario had to remember each child sim id; no way to navigate from parent to "the three branches that diverged at round 12"
- New endpoint `GET /api/simulation/<id>/lineage` — publish-gated, `Cache-Control: public, max-age=300`. Returns `{simulation_id, lineage_kind, parent | null, children: [...], total_children, counterfactual | null}`. Each child carries its own kind (fork / counterfactual) + optional `{trigger_round, label}` block so the badge renders "🔀 Counterfactual at round 12 (ceo_resigns)" inline. Children are public-only (private forks don't leak), sorted by `created_at` ascending (oldest fork first), capped at 50 entries
- New `backend/app/services/lineage_service.py` (~390 LoC, pure stdlib `json` + `os`): `MAX_CHILDREN=50` + `SCENARIO_PREVIEW_CHARS=80` constants, `_load_state` / `_load_config_scenario` / `_load_counterfactual` defensive disk readers, `_kind_for(state, cf)` discriminator (parent_id unset → original; parent_id + cf file → counterfactual; parent_id + no cf → fork), `_build_counterfactual_marker` extracts `{trigger_round, label}` for the badge, `_entry_for_sim(sim_id, sim_dir, state, include_kind)` composes a parent / child entry payload (scenario empty when parent unpublished), `_iter_candidate_sim_ids` walks data dir skipping dotfiles + non-dirs, `find_children(parent_id, data_dir)` scans the corpus for reverse pointers (skips self-pointers, private children, corrupt state), `build_lineage_payload(simulation_id, data_dir)` composes the response
