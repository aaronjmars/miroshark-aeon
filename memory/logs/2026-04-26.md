## Feature Built — Completion Webhook (PR #46)
- Picked from repo-actions Apr 24 idea #3 (Webhook Notification on Simulation Complete) — chosen over #1 (SSE), #2 (Leaderboard), #4 (Discord/Slack post button) because (a) strict superset of #4 once pointed at Zapier/n8n, (b) cleanest hook into the existing `send_push_notification` call site at runner completion, (c) fully backend-shaped so lower regression risk, (d) "webhook on completion" is the production-quality checklist item the Paradigm-adjacent ML/dev audience scans for.
- New `backend/app/services/webhook_service.py` (stdlib `urllib.request`, zero new deps): `build_payload` (mirrors ±0.2 stance threshold from share-card/gallery so consensus matches), `fire_webhook_for_simulation` (daemon-thread POST, `(sim_id,status)` dedup so the runner's exit-code path AND per-platform `simulation_end` event path can both call it), `send_test_webhook` (synchronous, used by Settings test button), `validate_url`/`mask_url` (http(s) only, mask to `scheme://host/***` so Slack/Discord secret never echoes back).
- Hooked into `simulation_runner.py` at both completion paths (line ~599 exit-code, line ~734 `simulation_end` event). Wrapped in try/except so a failing webhook never takes down the runner.
- `Config.WEBHOOK_URL` + `Config.PUBLIC_BASE_URL` env vars; `GET /api/settings` exposes `integrations.webhook = {configured, url_masked, public_base_url}`; `POST /api/settings` accepts the new fields with URL validation; new `POST /api/settings/test-webhook`.
- `SettingsPanel.vue` gains an "Integrations · Webhook" section above the MCP block with status badge, URL input (masked placeholder), optional public_base_url, and inline "Send test event" button.
- `docs/WEBHOOKS.md` (full payload schema, headers, delivery semantics, Slack/Discord/Zapier/n8n/custom-listener recipes); README Features + Documentation tables both updated; `.env.example` blocks for both env vars.
- 18 unit tests in `backend/tests/test_unit_webhook.py` covering payload shape (full/minimal/corrupt/long-scenario/failure-with-error), URL validation/masking, fire no-op when unconfigured, async dispatch (caller returns before slow POST), per-(sim_id,status) dedup including completed+failed both firing, exception swallowing, unknown-status guard, send_test_webhook end-to-end.
