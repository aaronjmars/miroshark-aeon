## Self-Improve — Tweet Allocator Bankr Diagnostics

- **Trigger:** `tweet-allocator` emitted `TWEET_ALLOCATOR_ERROR` two days running (Apr 25 + Apr 26) with the same notification text — "`.bankr-cache/verified-handles.json` is empty — `BANKR_API_KEY` secret is likely not set." That diagnosis is wrong: when `BANKR_API_KEY` is unset the prefetch script exits early WITHOUT creating the cache file, so an empty (vs missing) file means the prefetch ran but every Bankr lookup silently returned no wallet. Tweet-allocator was working Apr 22/23/24 with 5 wallets verified each day, so the secret IS set — real cause is more likely Bankr Agent API quota / rate-limit / transient outage. The bot was sending a loud daily error alert blaming the wrong root cause.
- **Picked over:** repo-actions Apr 24 idea #1 (SSE), #2 (leaderboard), #4 (Discord/Slack share button) — those are watched-repo work, not agent-self work. This one is purely agent-side and directly removes a recurring false-alarm notification.
- **Fix:**
  - `scripts/prefetch-bankr.sh` — adds a `write_status` helper that emits `.bankr-cache/prefetch-status.json` at every exit point. New fields: `status`, `note`, `timestamp`, `candidate_count`, `lookup_attempted`, `curl_failed`, `verified_count`, `null_count`. Status values: `no-api-key` (env var unset), `no-candidates` (zero handles to look up), `lookups-failed` (every Bankr call failed at curl/jobId — counts maintained inline via `LOOKUP_ATTEMPTED` / `CURL_FAILED`), `completed-no-wallets` (lookups succeeded but every handle returned null), `completed` (≥1 verified wallet). Also writes `{}` to `verified-handles.json` in the no-api-key branch so the skill sees a consistent "file exists, status sidecar tells you what happened" contract.
  - `skills/tweet-allocator/SKILL.md` step 4 — branches on the sidecar's `status` field instead of guessing from cache emptiness. Crucially: `no-api-key` becomes `TWEET_ALLOCATOR_DISABLED` and is **silent** (no notification — no recipient can fix it from the bot side); `lookups-failed` becomes `TWEET_ALLOCATOR_ERROR` with an accurate message ("Bankr Agent API unreachable (X/N curl failures). Check api.bankr.bot status / BANKR_API_KEY validity"); `completed-no-wallets` falls into the existing `_EMPTY` flow with a brief one-liner. `completed` continues to allocation as before. Status flags section updated to reflect the new four-state model.
- **Why this matters:** Yesterday's and today's alerts both told the operator to check `BANKR_API_KEY`, which would have been a wild-goose chase since the secret IS set. Going forward, if the same failure recurs, the alert will say "Bankr Agent API unreachable" with the actual lookup-failure count — a debugging signal instead of a false lead. And if `BANKR_API_KEY` ever does get rotated out, the daily noise stops entirely (TWEET_ALLOCATOR_DISABLED is silent).
