## Feature — aaronjmars/MiroShark
- **Built:** `wait` CLI subcommand
- **Why:** repo-actions 2026-06-22 idea #2 — `status` is a single snapshot and `cost` (#208) only means anything once a run ends, so integrators hand-rolled their own `while not done: sleep` poll loop around `cli status`. `wait` collapses that into one blocking call with correct exit codes (0 completed / 1 failed+stopped / 2 timeout), making `wait → report`/`wait → cost` scriptable. Serves priority-3 (scripted first runs) + ecosystem pipelines.
- **PR:** https://github.com/aaronjmars/MiroShark/pull/215
- **Files:** backend/cli.py (cmd_wait + wait subparser w/ --interval/--timeout), backend/tests/test_unit_cli.py (subcommand-set + test_wait_defaults_and_overrides), docs/CLI.md, docs/CLI.zh-CN.md
- **How:** argparse + urllib only, no new deps; reuses `_api` to poll `/run-status`, matches `runner_status` against terminal sets pulled from RunnerStatus enum (completed/failed/stopped); monotonic deadline, sleep capped to remaining time; progress → stderr so stdout stays clean for `--json`. No new HTTP surface → openapi drift test unaffected.
