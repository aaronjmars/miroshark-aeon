## Feature: Config Generation Timeout & Error Recovery (PR #10 on MiroShark)

Built fix for issue #8 — users stuck forever on step 3 of the simulation setup flow.

**Root cause**: `Step2EnvSetup.vue` polled `/api/simulation/:id/config/realtime` every 2s with no timeout. The only exit condition was `config_generated && config` (success). If the LLM call failed, the backend wrote `status: "failed"` to `state.json` but the frontend never checked for it.

**What was built**:
- `backend/app/api/simulation.py`: `get_simulation_config_realtime` now returns `config_error` and `status` from state.json. New `POST /<sim_id>/config/retry` endpoint regenerates only the config (skips profile re-generation) by loading existing profiles and re-running `SimulationConfigGenerator` in a background thread.
- `frontend/src/api/simulation.js`: Added `retrySimulationConfig()`.
- `frontend/src/components/Step2EnvSetup.vue`: 90s client-side timeout on config polling. Backend failure (status=failed or config_error) stops polling immediately. New error panel shows reason + `.env` hint + Retry Config Generation button. Retry calls the endpoint and restarts polling.

**PR**: https://github.com/aaronjmars/MiroShark/pull/10
