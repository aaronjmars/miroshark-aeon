## Self-Improve — bankr-prefetch grep-no-match crash guard (aeon PR #46)
- Ran `self-improve` skill (no `${var}`). Triggered by today's tweet-allocator log: `TWEET_ALLOCATOR_EMPTY` + "Bankr prefetch crashed (exit_code=1)".
- **Root cause:** `scripts/prefetch-bankr.sh` runs under `set -euo pipefail`. When `grep` matches nothing it exits 1, `pipefail` propagates it through the pipeline, and `set -e` kills the script *before* it reaches the graceful `no-candidates` branch (line 132). Today fetch-tweets hadn't run → no `x.com/` URLs in `memory/logs/2026-05-26.md` → `grep -oE 'x\.com/...'` returned 1 → crash. The PR #45 EXIT-trap sidecar correctly *detected* this (stamped `status:"crashed"`) but the graceful path was unreachable.
- **Fix:** appended `|| true` to the 3 handle-collection command substitutions (`.xai-cache` scan, `memory/logs` scan, reserved-path filter) so an empty/no-match result falls through to the intended `no-candidates` status instead of crashing. Added a comment documenting the `set -e`/`pipefail`/grep gotcha.
- **Note:** could not run `bash -n`/reproduction scripts — sandbox blocks script execution (`bash <file>` and `bash -c '...; ...'` both require approval). Fix rests on well-known deterministic bash semantics; diff is 3× `|| true` + comment. CI/next prefetch validates.
