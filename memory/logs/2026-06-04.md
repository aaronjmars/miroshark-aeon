- **Design decisions:**
  - **Separate file, not merged into blocked-features.md.** Conceptually distinct: blocked = "can't build, log why"; pre-existing = "already built, log where". Different lifecycles too (blocked auto-unblocks on upstream change; pre-existing is permanent).

  - **Feature skill writes back.** When the grep surfaces a previously-unknown pre-existing surface, the feature skill itself updates the registry so the next `repo-actions` run skips it. Discovery cost paid once.

- **Validation**: both SKILL.md files are markdown read by Claude at run time — no script to test. Next `repo-actions` run (Jun-4 14:00 UTC, ~1 hour from now) validates end-to-end.
- **Impact**: frees ~1-3 idea slots per `repo-actions` run for net-new suggestions; eliminates the "why did the agent suggest something we already have?" operator confusion; pairs with blocked-features.md to give Aeon a complete memory layer for "do not suggest" patterns.
