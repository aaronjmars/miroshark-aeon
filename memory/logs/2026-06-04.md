Updated `skills/repo-actions/SKILL.md` step 4 to read both registries (block + pre-existing), with distinct exclusion notes (`Excluded (blocked):` vs `Excluded (pre-existing):`) so the operator can tell why each idea was filtered. Updated `skills/feature/SKILL.md` step 6 to check the registry before the upstream grep (cheap short-circuit) and to **add new entries** when its grep surfaces a pre-existing surface not yet in the registry (so the cost of each discovery is paid once).
- **Files changed (3):**
  - `memory/topics/pre-existing-features.md` — new file. Bootstrapped with 8 entries extracted from MEMORY.md "Next Priorities" and dated `memory/logs/` verifications.
  - `skills/repo-actions/SKILL.md` — step 4 extended to read pre-existing-features.md alongside blocked-features.md.
  - `skills/feature/SKILL.md` — step 6 reads pre-existing-features.md before the grep, and writes back new entries when grep discovers a pre-existing surface.
- **Design decisions:**
  - **Separate file, not merged into blocked-features.md.** Conceptually distinct: blocked = "can't build, log why"; pre-existing = "already built, log where". Different lifecycles too (blocked auto-unblocks on upstream change; pre-existing is permanent).
  - **No auto-removal.** Features don't unship — once verified, the entry stays forever. Simpler lifecycle than blocked.
  - **Feature skill writes back.** When the grep surfaces a previously-unknown pre-existing surface, the feature skill itself updates the registry so the next `repo-actions` run skips it. Discovery cost paid once.
  - **Distinct exclusion notes in articles.** `Excluded (blocked):` vs `Excluded (pre-existing):` — the operator can see at a glance whether an idea was filtered because it can't be built or because it already is.
- **Validation**: both SKILL.md files are markdown read by Claude at run time — no script to test. Next `repo-actions` run (Jun-4 14:00 UTC, ~1 hour from now) validates end-to-end.
- **Impact**: frees ~1-3 idea slots per `repo-actions` run for net-new suggestions; eliminates the "why did the agent suggest something we already have?" operator confusion; pairs with blocked-features.md to give Aeon a complete memory layer for "do not suggest" patterns.
