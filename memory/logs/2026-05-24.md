## Self-Improve — bankr-prefetch EXIT-trap crash sidecar
- **Aeon PR**: https://github.com/aaronjmars/miroshark-aeon/pull/45 (branch `improve/bankr-prefetch-crash-trap`)
- **Trigger**: today's `tweet-allocator` run logged `TWEET_ALLOCATOR_ERROR — prefetch-bankr.sh did not run; .bankr-cache/prefetch-status.json is missing` and fired a workflow-misconfiguration alert. Workflow's `for script in scripts/prefetch-*.sh` loop **does** invoke the script; `set -euo pipefail` exits silently if any of the early jq/grep pipelines bail before reaching a `write_status` call. The skill then can't tell "workflow misconfigured" (file truly absent) from "script crashed mid-run" (silent failure).
- **Pre-existing branches/PRs check**: `gh pr list --state open` returned `[]` on miroshark-aeon — no conflicting in-flight work.
- **Files changed**:
  - `scripts/prefetch-bankr.sh` (+25): `trap ... EXIT` set right after `mkdir -p .bankr-cache`. Fires only when `$? != 0` AND `prefetch-status.json` is absent — stamps `{status:"crashed", exit_code:N, note, timestamp}` via jq. Normal exits (which always write via `write_status`) are untouched. Early `exit 0` for non-tweet-allocator skills happens **before** the trap registers, so those still skip cleanly.
  - `skills/tweet-allocator/SKILL.md` (+3/-2): new `"crashed"` branch in the status switch with its own alert message (carries the exit code from the `note` field); `Status flags` section updated to list `crashed` among `TWEET_ALLOCATOR_ERROR` variants.
- **Impact**: next silent prefetch crash produces an actionable diagnostic (exit code + timestamp) instead of a misleading "workflow misconfigured" alert. Truly-absent file still falls through to the original missing-file branch.
