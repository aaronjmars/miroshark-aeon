### Endpoint added
- `GET /watch/<simulation_id>` — self-contained server-rendered HTML page built for live spectating. Vanilla-JS poller updates the belief bar, round counter, and progress bar every 15 s by hitting the existing `/api/simulation/<id>/embed-summary` and `/api/simulation/<id>/run-status` REST endpoints. Once the runner reaches a terminal state polling stops and the "View full simulation →" + "Fork this scenario →" CTAs are revealed. OG / Twitter-card meta tags embedded in the body so a tweet of `/watch/<id>` auto-unfurls as a 1200×630 image card the same way `/share/<id>` does. `og:description` becomes `Round N/M · Bullish X% · Neutral Y% · Bearish Z% — watch live.` for in-flight runs, falls back to bare scenario for idle, generic string when nothing published. `Cache-Control: public, max-age=60`. Mounted at the root via `watch_bp` — no `/api` prefix, mirroring `share_bp` pattern (PR #42).

### Files
- **NEW** `backend/app/services/watch_renderer.py` (~600 LoC) — pure stdlib (`html` + `json`). `STANCE_THRESHOLD = 0.2` matches every other surface so a spectator who sees the share card on Twitter and clicks through to `/watch/<id>` doesn't see the numbers shift mid-flow. `_truncate` preserves trailing ellipsis under OG char budgets (200 for title, 280 for description, 220 for SSR scenario header). `_belief_summary` graceful-degrades on corrupt input (returns `(0,0,0)`). `_broadcast_js()` emits the in-page poller as a single triple-quoted string — 15 s cadence, 60 s back-off on transient failures, 6 h absolute timeout, one trailing refresh after 4 s once terminal so the final belief snapshot lands. `_build_meta_description` composes the OG description from round + split + scenario. `_build_initial_state` produces the bootstrap JSON blob. `render_watch_html` assembles the full document — OG / Twitter tags, inline CSS (~200 lines, gradient dark theme + pulsing live badge + responsive @media queries), SSR belief bars with proportional widths matching the legend numbers, bootstrap blob in a `<script type="application/json">` tag, vanilla-JS poller in a final `<script>` tag.
