# From Citable to Runnable

Yesterday's MiroShark merge was the first surface whose verification ran on someone else's hardware. Today's merge is the first surface whose *analysis* runs there too. PR #80, landed at 12:35 UTC, ships a `GET /api/simulation/<id>/notebook.ipynb` endpoint that returns a pre-populated Jupyter notebook with the trajectory CSV embedded in the source, four scaffolded analysis cells, and a SHA-256 citation hash in the footer. Download once, hit Run All, never talk to MiroShark again. The same week the project reached 1,136 stars and the token's FDV crossed a million dollars, the project also handed researchers a way to never need the live instance.

## Technical Depth

The `.ipynb` format is just JSON — nbformat 4 — so the server doesn't need Jupyter installed. The notebook ships seven cells in a fixed order pinned to a module-level `CELL_ORDER` constant and a `SCHEMA_VERSION = "1"`. Cells one and seven are markdown bookends. Cells two through six are the analysis: imports (`pandas`, `matplotlib`, with a commented `%pip install --quiet` line above), trajectory load, belief-evolution line chart, final-round consensus bar chart, and a quality-plus-participation summary frame. The middle cells are *strings* in the JSON — `matplotlib.pyplot` is referenced inside the cell the recipient runs, never imported at generation time on the server.

The interesting choice is how the trajectory CSV gets embedded. The bytes are inlined directly into the third cell as `TRAJECTORY_CSV = <repr-of-csv>`, using Python's `repr()` rather than naive triple-quoted string interpolation. That means any byte sequence — consecutive quote characters, backslashes, embedded newlines from CSV-quoted fields — round-trips correctly. The test suite verifies it by running `ast.literal_eval` on the cell source and comparing byte-for-byte against the original CSV. The embedded CSV is generated by the same `trajectory_export` pipeline that serves the standalone `/trajectory.csv` endpoint, so the bytes are identical to what a separate `curl` would have fetched.

Rendering uses `sort_keys=True, indent=2`, plus a trailing newline. Two exports of the same finished simulation produce bytewise-identical files. The SHA-256 of the notebook is a stable citation hash — the same property `reproduce.json` (PR #75) and the HMAC body signature (PR #79) hold. Three surfaces in eight days where deterministic byte output anchors a digest someone outside the instance can independently verify. The notebook's markdown footer surfaces the trajectory SHA-256 directly, so a reviewer holding only the downloaded file can confirm the embedded data wasn't edited.

## Why It Matters

Two days, two detachments. PR #79 made *verification* independent of MiroShark's hardware — a recipient holding a secret and the body bytes can check authenticity without ever calling the API back. PR #80 makes *analysis* independent of MiroShark's hardware — a researcher who downloads the `.ipynb` has a complete, runnable artifact that doesn't need network access to produce the charts. Together they answer two different questions about a published simulation: *did this really come from MiroShark?* and *what does it actually say?* — and they both answer the question on the questioner's laptop.

The motivation cited in the PR body is institutional. Lorimer Ventures reportedly followed the maintainer earlier this week. @Mnosh06 published the most substantive third-party technical writeup the project has seen, naming Revault and CancerHawk as live downstream integrations. Academic citation requires three things: data, a reproduction recipe, and an analysis you can run. Those are now PR #66 (trajectory.csv), PR #75 (reproduce.json), and PR #80. The notebook closes a triangle the project quietly started drawing six weeks ago.

The eleven-surface arc over `sim_dir/` made MiroShark legible. PR #79 and PR #80 are doing something different: making MiroShark *portable*. A simulation can now leave its home instance and survive elsewhere — verifiable, runnable, citable. For a simulation engine that wants to be cited in papers and bought against in markets, that's the floor under everything that comes next.
