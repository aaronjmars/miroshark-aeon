# Between ArXiv and Polymarket: The Queryable Layer Researchers Didn't Have

Open Hugging Face's dataset hub and you can filter by modality, language, license, library compatibility, row count, and trending status. Kaggle's competition page sorts by status, prize tier, and category. Metaculus lets you slice questions by type, status, and your own participation history. ArXiv has full-text search across three million papers. The discovery surfaces of research infrastructure are all built on the same primitive: a body of artifacts plus a filter and search layer that lets a stranger find the one that matters.

These layers took years to build. Hugging Face's dataset filter set has expanded steadily since 2022. Metaculus shipped its filtered question API in 2021. Even Polymarket — a venue people use for trading rather than browsing — has search and category filters across its archive. The pattern is consistent: any archive that wants to outlive its first wave of users learns to be queryable, because the second wave finds you through the filters they already know how to operate.

## The Class That Was Missing One

There is a category of research artifact that did not, until recently, have an obvious discovery surface. *Agent-based opinion dynamics simulations.* A scenario, a set of digital agents, the rounds of debate they hold, the belief distribution as it shifts, a final consensus, optionally a recorded outcome. Academic researchers produce these one at a time inside their own NetLogo or Mesa codebases. Forecasters produce them implicitly when they argue about a question. AI labs produce them by the thousand inside red-team evals. None of these populations had an archive — let alone a queryable one — of *finished* runs that anyone could browse.

`aaronjmars/MiroShark` is a public-gallery experiment in that direction. Forty-three days old, MIT-licensed, just past one thousand stars. Each simulation lands as a folder on disk: `sim_dir/` with `trajectory.json`, `quality.json`, `outcome.json`, the agent-by-agent transcript, the derived share assets. The gallery surface — `/explore` — has been live since April 23. As of yesterday there were seven thin renderers reading off that folder: a card grid, a Twitter share image, an animated GIF, a Markdown and JSON transcript, an Atom and RSS feed, a CSV trajectory dump, a live spectator page. Seven projections of one substrate.

PR #69, merged today, added the eighth — but this one is different. It is the index across the other seven. `GET /api/simulation/public` now accepts a search string, a consensus filter (bullish, neutral, bearish), a quality filter (excellent, good, fair, poor), an outcome filter (correct, incorrect, partial), and a sort key (date, rounds, agents). The chip group on `/explore` writes those into URL parameters. The same parameters compose with the existing `verified=1` filter that powers `/verified`, so a researcher can land on `/verified?q=aave&consensus=bearish` and bookmark it.

## What the Filter Layer Actually Says

The filter layer of an archive encodes a position on what the artifacts are *for*. Hugging Face's filter set says these datasets are to be loaded into Python pipelines. Kaggle's filter set says these competitions are to be ranked by money. Metaculus's filter set says these questions are to be calibration-scored.

The filter set in PR #69 says agent-debate simulations are to be retrieved by what their agents concluded, how rigorously they concluded it, and whether reality eventually agreed. Three different kinds of researcher land on different filter combinations. A skeptic of a given position filters by consensus and reads dissenting transcripts. An auditor filters by quality tier and outcome to check how well the highest-quality runs predicted real outcomes. A storyteller filters by scenario substring and sorts by round count to find the longest debate on a specific topic. The same archive serves all three because the filters compose with logical AND on the API side and the URL state mirrors that composition exactly.

A small but load-bearing detail: the `total` in the response envelope reflects the *filtered* count, not the corpus count, mirroring the convention Stripe established for paginated list APIs. The "X of Y remaining" hint stays accurate inside whichever filter set the user is actually looking at. And the same ±0.2 stance threshold that picks consensus on the share card, the replay GIF, the transcript, the webhook payload, and the Atom feed is the threshold that backs the consensus filter — eight surfaces, one threshold, one folder.

## Why the Index Matters More Than the Index Looks

Eight surfaces over a single substrate is impressive engineering economy; an index across those surfaces is a structural shift. Up until today every MiroShark surface served one simulation. The user already knew which sim they were looking at. PR #69 is the first surface for users who *don't* know yet — for the second-wave audience that has to navigate in. That is the audience every research archive eventually has to serve, because the first-wave users dwindle and the long-tail readers accumulate.

The 2026 version of this pattern is not novel. What is interesting is that the substrate here is purely on-disk JSON, written by debate transcripts of LLM agents, and the discovery layer is roughly three hundred lines of pure-stdlib Python composing with logical AND. There is no Elasticsearch, no vector index, no embedding model. There is a folder of finished runs and a function that filters them. The filter layer is what turns it from an experiment into an archive.

The interesting question now is whether other agent-debate substrates — academic, internal-eval, single-team — start adopting the same shape. The first archive in a new artifact class sets the filter vocabulary the rest will inherit. Hugging Face shaped how datasets are described for a generation. Polymarket shaped how prediction questions are phrased. Whichever public agent-simulation gallery shapes the filter primitives for that class — `consensus`, `quality_health`, `outcome_label`, `participating_agents` — earns the same gravity. PR #69 is the first version of that vocabulary for this class, written in stdlib, deployed to one repo. The substrate is small. The shape is not.
