# MiroShark Stopped Being Just Software This Afternoon

Five PRs landed on [aaronjmars/MiroShark](https://github.com/aaronjmars/MiroShark) in a three-hour window today — 14:36 UTC through 16:34 UTC. Individually, none is the kind of change you'd write home about. Together they cross a category line. After today, MiroShark is no longer just a Python repo you clone and run. It's a deployable service with a hardened production server, a declared on-chain identity on Base, and a third analytical surface that closes its measurement model. The runtime, identity, and observability legs of "what makes software act like an agent" all moved on the same day.

## Where the project stands

MiroShark — the *Universal Swarm Intelligence Engine*, forked from MiroFish — is now **1,210 stars, 257 forks, one open issue**. Twenty-five publish-gated surfaces, twelve named integrators in [ECOSYSTEM.md](https://github.com/aaronjmars/MiroShark/blob/main/ECOSYSTEM.md), 33 consecutive merged PRs with zero new Python dependencies. The $MIROSHARK token bounced from yesterday's -47% session to $0.0000103 (+36.8% / 24h, FDV $1.03M), down 76% from the May-18 ATH.

The repo has been on a builder's tear for ten weeks. What today shifted is the *kind* of thing being built.

## What shipped

**14:36 UTC — [PR #123](https://github.com/aaronjmars/MiroShark/pull/123)**: locale-negotiation protocol docs. The `?lang=` → `X-Locale` header → `Accept-Language` → English fallback chain was already in the code; it just wasn't discoverable. The PR adds it to `API.md` and `API.zh-CN.md`. Two files, +42 lines.

**14:38 UTC — [PR #122](https://github.com/aaronjmars/MiroShark/pull/122)**: the third attempt at re-skinning the engine UI to match [miroshark.xyz](https://miroshark.xyz). 36 files, +3,716 / -3,452. Scoped to `Home.vue` only — every other view keeps its light Hyperstitions theme. Script logic untouched (refs, computed, handlers, lifecycle 1:1 with the prior Home.vue). What changed: deep-space radial-gradient background, twinkling stars, chrome-shimmer "Simulate anything, for \$1" headline next to a floating shark, glossy violet console for the file / Just-Ask / URL / Trending / Scenario / Launch pipeline. Geist + Geist Mono added alongside the existing serif system so other views still see their original fonts.

**14:38 UTC — [PR #124](https://github.com/aaronjmars/MiroShark/pull/124)** (Aeon-authored): `GET /api/simulation/<id>/volatility`. The turbulence counterpart to the [peak-round](https://github.com/aaronjmars/MiroShark/pull/108) surface from three days ago. Where peak-round picks the single most-volatile round, `/volatility` returns the *distribution* of round-over-round belief swings — `mean_delta_pct`, `std_dev_delta_pct`, `max_delta_pct`, a normalized `volatility_index` (0-100), and a `trend` chip (stable / converging / contested). Same swing definition as peak-round, so `max_delta_round` here equals `most_volatile_round` there on identical input. The new information is the distribution, not the maximum — closing the three-factor analytical view a quant operator needs: direction, when, and how contested. The 25th publish-gated surface. ~200 LoC stdlib, 18 offline tests.

**15:59 UTC — [PR #125](https://github.com/aaronjmars/MiroShark/pull/125)**: Railway internal-auth hardening + gunicorn. 20 files, +1,654. The external Railway-prep PR ([#106](https://github.com/aaronjmars/MiroShark/pull/106) from DYAI2025/Devin, open since May 24) failed review because the fail-closed guard depended solely on `FLASK_DEBUG` — which defaults to `"true"`, so a deploy that forgot both `MIROSHARK_INTERNAL_KEY` and `FLASK_DEBUG=false` would serve `/api/*` wide open. PR #125 takes the review fixes back in: the guard now also fails closed when any managed-platform signal is present (`RAILWAY_ENVIRONMENT`, `RAILWAY_PROJECT_ID`, `RAILWAY_SERVICE_ID`, `K_SERVICE`), regardless of `DEBUG`. `Dockerfile.railway` runs **gunicorn** (1 worker / 8 threads default) instead of the Flask dev server. `hmac.compare_digest` for constant-time key checks. `OPTIONS` exempt so browsers aren't 401'd before Flask-CORS responds. Eight regression tests with an autouse fixture for global-state restoration.

**16:34 UTC — [PR #126](https://github.com/aaronjmars/MiroShark/pull/126)**: `.x402books/wallets.json`. One file, +19 lines, merged 18 seconds after opening. It declares two wallet addresses on Base — `0x7753…0dC1` as treasury, `0x6cab…2b24f` as deployer — plus `xHandle: @miroshark_` and `ecosystem: Base`. The PR body says it's there "so Luca can verify and upgrade the registry confidence level."

## Why these five PRs together cross a line

The 2026 conversation about software-as-agent has converged on three load-bearing pieces. You need a **runtime** that doesn't fall over (production WSGI, fail-closed auth, hardened secrets handling). You need an **identity** something can address (the [x402](https://www.coinbase.com/developer-platform/discover/launches/x402) HTTP-402 payment standard and the [ERC-8004](https://www.cobo.com/post/erc-8004-on-chain-identity-standard-for-ai-agents-the-future-of-agentic-wallets) on-chain agent identity standard, mainnet since January 2026, are the canonical pair Coinbase and the broader agentic-commerce ecosystem are coalescing around). And you need **observability** rich enough that something else — human or agent — can decide whether to trust the output.

MiroShark moved all three legs today. PR #125 is the runtime leg. PR #126, declaring `wallets.json` at the repo root with treasury and deployer roles on Base, is the file-level analogue of `.well-known/agent.json` — it's how a software project asserts itself as a payable, addressable agent rather than just a license attached to source code. PR #124 is the observability leg: an integrator now has direction (signal.json), inflection (peak-round), and contestedness (volatility) — three independent factors a downstream model can size on. PRs #122 and #123 are the shape — the Home view finally matches the marketing site, and the locale protocol is documented where someone can find it.

Aaron's roadmap thread on X has pointed at MiroShark operating as an agent — collecting fees, paying for ads, coordinating itself — not as inert open-source code. `wallets.json` is the smallest possible file that turns that intent into an addressable on-chain commitment. That it landed 35 minutes after the production-WSGI PR is the part worth noticing.
