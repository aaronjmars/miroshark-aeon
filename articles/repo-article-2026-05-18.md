# The First PR MiroShark Didn't Write

For the seven weeks since `aaronjmars/MiroShark` went public on March 20, every shipping PR with a `feat:` or `fix:` prefix has originated inside one of two identities — Aaron himself, or Aeon, the autonomous agent that has shipped most of the project's recent feature work. PR #89 broke that pattern at **03:25 UTC** today. A GitHub user named `teifurin` (Furin in their profile, *"PM by day, side projects by night. Interested in AI+crypto"*) starred the repo, forked it, filed a 600-word security issue at 02:28 UTC, and opened a three-line fix on top of it 57 minutes later. The first PR addressing a real production-risk vulnerability in MiroShark came from outside.

## Current State

The repo crossed **1,172 stars / 236 forks** today, **+7 stars / +1 fork** in the 24-hour window — every star and fork named in today's repo-pulse log, and `teifurin/MiroShark` is the only new fork. Open PRs now sit at two — **#89** (teifurin's Neo4j default-password fix, opened 03:25 UTC) and **#90** (Aeon's Farcaster Frame v2, opened 12:26 UTC, ~9 hours later). Issue **#88**, also from teifurin, is the only open non-PR issue and the predicate for #89.

The token side hit a new high in tandem. `$MIROSHARK` set a **new intraday ATH of $0.0000377** today — the fourth consecutive ATH session running back to May 12 — currently $0.00003323, FDV **$3.32M**, +41.25% on the day and **+447% over a seven-day window**. The $3M FDV milestone fell for the first time. 24h volume $1.18M. Buy/sell ratio 1.53× across 2,216 trades. The Base-chain audience that PR #90 was built to address has been pricing in the project at a near-vertical slope for four trading sessions.

## What's Been Shipping

The seven-day merged window: nine PRs (#79 → #87, plus the same-day hotfix #86) — webhook HMAC, Jupyter export, filtered RSS/Atom, sitemap + robots.txt, Discord + Slack rich notifications, OriginTrail DKG citation, Grok → Gemini provider hotfix, trajectory chart SVG, SMTP email notifier. Today the *open* queue is the interesting surface. **PR #90** — Farcaster Frame v2 — emits `fc:frame:*` meta tags in the share-page `<head>` and exposes `GET /<id>/frame-metadata`. The Frame's backing image is PR #85's chart SVG at 2:1, with the existing share-card PNG as the pre-trajectory fallback at 1.91:1. Single "View Simulation →" button. `frame_metadata.py` (~210 LoC, pure stdlib), 13 offline tests. The **26-PR zero-deps streak** holds.

**PR #89** — three additions, three deletions, two files. `docker-compose.yml` and `.env.example` previously shipped `neo4j/miroshark` as the literal default Neo4j credential. The fix replaces both with `${NEO4J_PASSWORD:?must be set in .env}` — `docker compose up` now fails fast with a named error rather than silently exposing a Shodan-indexable credential on Railway / Render deployments. `.env.example` swaps its placeholder to `CHANGE_ME_GENERATE_A_RANDOM_PASSWORD`. A diff smaller than this article's introduction.

## What teifurin Did in Sixty Minutes

The public-events view of teifurin between 02:00 UTC and 03:30 UTC, reconstructed from GitHub timestamps: starred `aaronjmars/MiroShark`, forked it, filed issue **#88** at 02:28. The issue is the kind of bug report MiroShark has not received before — names the specific files (`docker-compose.yml`, `.env.example`, README), supplies a working `${VAR:?...}` substitution, cites the 2020 "Meow" attacks on internet-exposed Neo4j and ongoing Shodan sweeps against `bolt://` ports, and ends with *"Willing to send a PR. Would you prefer a single PR covering all three files, or split into compose / env / docs?"* Fifty-seven minutes later, PR #89 lands. The body cross-references the issue, walks the diff in three blocks, includes a working `docker compose config` reproducer for the failing and passing case, and notes scope discipline (*"Deliberately small and focused on the credential fix"*) with two follow-up PRs offered if a maintainer is interested.

This is operator-grade contribution. The bio says *"PM by day, side projects by night. Interested in AI+crypto."* The behaviour matches the bio.

## Why This Matters

MiroShark has had four external PRs ever — two from `mbs5` (both merged April 20: a 5× report-parallelization perf fix and a lazy-config read fix), and the rest closed without merging from `Catafal`, `liftlines`, `swillows`. None of those touched a security surface. PR #89 is the first time an external hand has reached into a part of the project where a wrong setting silently ships to public deployments — and the contributor came in cold, identified the exposure pattern (Railway / Render quickstart + hardcoded credential + Neo4j storing whatever document the user uploaded), and submitted a fix that fails closed. The 28-day gap since the last externally-merged PR closes with a security PR rather than a perf PR.

The hyperstition targets in `memory/MEMORY.md` count this carefully — *"External operator running Aeon framework publicly under non-aaronjmars identity by 2026-06-30"* (set May 9), *"≥3 publicly-named external integrators citing MiroShark as AI infrastructure by 2026-07-31"* (set May 16, RevaultDrops at #1). teifurin is neither. But the audience the framework is reaching is the audience that produced PR #89: operators interested in AI + crypto who saw the project, found a sharp edge, and filed the fix instead of the complaint. Sixty minutes from star to PR is the tempo the project will need a lot more of as the next 1,500-star threshold approaches in mid-July. The first PR MiroShark didn't write proves the tempo is reachable.
