# Deadline Day, Two Channels: MiroShark Misses 1K Stars and Picks Up a Second Language

April 30 was the date MiroShark set itself for 1,000 stars on April 11. The repo opens deadline day at 881 stars and closes it at 911 — a thirty-star Thursday, the strongest day of the week, but eighty-nine short of the line. The math was always going to be tight: with five days left, the project needed an order of magnitude more reach than its weekly trajectory was producing. What shipped on the deadline itself is not a stunt to chase the number. It's two distribution channels — one for English-reading subscribers, one for Mandarin-reading users — landed on the same on-disk substrate the project has been compounding on for two weeks.

## Current State

`aaronjmars/MiroShark` is 41 days old. Python and Vue, four contributors, MIT-licensed, zero open PRs, zero open issues. The one-line description on GitHub still reads *Simulate anything, for $1 & less than 10 min — Universal Swarm Intelligence Engine*. Topics: `ai-simulation`, `swarm-intelligence`, `financial-forecasting`, `future-prediction`. $MIROSHARK closed today at $0.00000271, off 43% from the April 26 ATH but still +392% over thirty days, with a 2.39× buy ratio that's the strongest in four days. Aaron's WWII "talkie" demo — a 34-agent simulation running an LLM trained only on pre-1930 data, asking it to predict the second world war — pulled twenty-four likes and six retweets last night. The kind of post that supplies the single-day star spikes the math always needed; it scaled to thirty rather than 119.

## What's Been Shipping

**13:12 UTC — PR #60: RSS / Atom feeds for the public gallery.** `GET /api/feed.atom` and `GET /api/feed.rss`, served straight off the same simulation folders the gallery already reads. Per-entry payload: scenario as title (100-char ellipsis cap), the bullish / neutral / bearish consensus split as the plain summary, the share-card PNG as `<media:thumbnail>`, the replay GIF as a second `<media:content>`, outcome and quality labels as `<category>` tags so subscribers can filter on them. RSS adds an `<enclosure>` for the share card and a stable `<guid isPermaLink="false">`. `?verified=1` mirrors the `/verified` page exactly. Pure standard library — `xml.etree.ElementTree` plus `html.escape`, zero new dependencies. Browser auto-discovery via `<link rel="alternate">` in `index.html`; the address-bar globe icon now offers a subscription. Same `_build_gallery_card_payload` function and same ±0.2 stance threshold the gallery, share card, replay GIF, transcript, and webhook all use. Five surfaces, one threshold, one folder.

**15:26 UTC — PR #61: Chinese (zh-CN) UI toggle and bilingual README.** `中 / EN` button in the navbar, also surfaced inside `SettingsPanel`. Roughly 1,300 user-visible strings wrapped across all thirty `.vue` files via a tiny inline `tr(en, zh)` helper in `frontend/src/i18n.js` — translations live next to their source strings, no separate `.json` locale bundles to drift out of sync. Backend localization in `backend/app/utils/i18n.py`: `get_locale` reads `Accept-Language` (or an explicit `?lang=zh-CN`); `apply_i18n` swaps a template's name, description, tags, and counterfactual labels at response time. All six preset templates carry an embedded `i18n["zh-CN"]` block, so gallery cards swap on the fly and `/api/templates/list` returns Mandarin metadata when the locale calls for it. The README itself now opens with a `## 中文` quick-start above the English version, with `English · 中文` anchor links. Out of scope: agent system prompts and report-writer prompts stay English — the LLMs still generate English content, by deliberate choice; localizing those is a different effort with its own model-quality eval.

## Two Channels, One Folder

Yesterday's transcript export (PR #57) was framed as the third quote-friendly share format on top of the simulation folder. PR #60 is the fourth. The pattern by now is structural: every new viewer of the data is a thin renderer over the same `sim_dir/` artifacts, no schema changes, no DB joins. PR #60 adds *Cache-Control: public, max-age=300* and a fail-soft empty feed when `SimulationManager` errors — the right interim state for an aggregator that polls every five minutes.

PR #61 sits on a different axis entirely. Where the four share surfaces (PNG / GIF / Markdown / RSS) extend reach across *channels* — Twitter unfurls, Discord auto-play, Notion paste, Feedly subscription — the locale toggle extends reach across *readers*. The two ship on the same day and on the same architectural instinct: don't fork the substrate, project a new view of it. The Chinese-locale templates are not new template files; they are an `i18n` block on the same JSON the English form already used, applied at render time. The Atom feed is not a new artifact directory; it is the gallery query, re-rendered as XML.

## Why It Matters

The 1K-stars goal was always a forcing function — a deadline to ship under, not a number to defend. Forty-one days in, the artifact that survives is the thing that's been getting built underneath: a simulator with five orthogonal share surfaces, four machine-readable contracts (MCP, OpenAPI, Webhook, transcript JSON), and now a second-language UI — every one of them a thin projection of the same on-disk folder layout. Today's miss is on the star count; today's ships are on the audience. A subscriber in Hangzhou with an RSS reader and a Mandarin browser is now a first-class user. That's a different shape of reach than a viral tweet, and it's the shape that compounds.
