## aaronjmars/MiroShark

### Theme 1: Production runtime + on-chain agent identity
**Summary:** Two follow-ups that close out the runtime and identity legs of the "what makes software act like an agent" stack. PR #125 stops trusting `FLASK_DEBUG` as the only signal for fail-closed auth and switches the container from Flask's dev server to gunicorn; PR #126 declares MiroShark's two on-chain wallets in the standard x402books registry format so an external maintainer can verify them. Both built on infrastructure work from earlier the same afternoon: PR #125 incorporates review fixes back from the long-stalled external PR #106 (Railway deployment), and PR #126 formalizes a long-running roadmap thread.

**Commits:**
- `177591f` / `375ef84` — `fix: harden internal-auth guard + run production WSGI server (#125)`
  - `backend/app/__init__.py` (+33, −8) — the meat of the fix. Adds a new `_is_deployed_environment()` helper that returns `True` whenever any of `RAILWAY_ENVIRONMENT`, `RAILWAY_PROJECT_ID`, `RAILWAY_SERVICE_ID`, or `K_SERVICE` (Cloud Run) is set. The internal-key guard now fails closed when *either* `not Config.DEBUG` *or* `_is_deployed_environment()` is true — previously it only checked `Config.DEBUG`, and `FLASK_DEBUG` defaults to `"True"`, so a deploy that forgot both the key and `FLASK_DEBUG=false` would serve `/api/*` openly. Also: switches the key comparison from `provided_key != internal_key` to `hmac.compare_digest()` (constant-time, kills the timing oracle), and adds an early-return for `OPTIONS` so CORS preflight isn't 401'd before Flask-CORS can answer.
  - `Dockerfile.railway` (+22, −3) — replaces `python backend/run.py` (the Flask dev server) with `exec gunicorn`, single worker / 8 threads default, `gthread` worker class. Single worker is deliberate: the backend loads torch + sentence-transformers per worker so extra workers multiply memory, and the workload is I/O-bound (LLM + Neo4j calls) so threads handle it well. `GUNICORN_WORKERS` / `GUNICORN_THREADS` / `GUNICORN_TIMEOUT` env vars exposed for platform overrides; `exec` ensures gunicorn is PID 1 so SIGTERM lands on it for graceful shutdown.
  - `backend/pyproject.toml` (+2) + `backend/uv.lock` (+15, −1) — adds `gunicorn` as a runtime dep.
  - `cloudrun.env.yaml.example` (new file, +41) — fills a previously missing template (deploy_cloudrun.sh required `cloudrun.env.yaml` but no example existed).
  - `railway.env.example` (+8, −5) — stops hardcoding `PORT` (platform injects it) and adds a comment explaining why `FLASK_DEBUG=false` is mandatory.
  - `backend/tests/test_unit_internal_auth.py` (+64, −17) — autouse fixture restores mutated module-global state between tests (was leaking under pytest's collection order), plus a new regression test that fakes `RAILWAY_ENVIRONMENT=production` with `FLASK_DEBUG=true` and asserts the guard still 503s — the exact case the original code missed.

- `4a71e20` — `Add .x402books/wallets.json to declare agent wallets (#126)`
  - `.x402books/wallets.json` (new file, +19) — the entire diff. Declares MiroShark's two on-chain wallets in the x402books registry schema: agent name `MiroShark`, x handle `@miroshark_`, ecosystem `Base`, and a two-element `wallets` array. Treasury (`0x7753…0dC1`) and deployer (`0x6cab…2b24f`), both on Base, both annotated by `role`. PR body cites "Luca" maintaining the registry — this is the declaration side of an external verification pipeline. The PR opened and merged 18 seconds apart with zero code review delay, which reads like a known-good drop into a known schema rather than something that needed iteration.

**Impact:** PR #125 closes the realistic deploy-misconfiguration failure mode (`FLASK_DEBUG` left at its default) and removes the timing oracle on the internal key. The gunicorn switch turns Railway / Cloud Run deploys from "the Flask dev server in production" into something that can take real traffic. PR #126 makes MiroShark's on-chain identity machine-verifiable through the standard x402books registry — third parties can now check "does the wallet that minted MIROSHARK match the one this agent self-declares" without trusting any single source. Combined with the existing ERC-8004 work, these are the runtime + identity halves of the agent stack catching up with the analytical work shipped earlier in the same afternoon.
