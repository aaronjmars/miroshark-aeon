# Week in Review: CLI Has a Full Lifecycle, Engine Handles Real-World LLMs

*2026-06-29 — Weekly shipping update*

> Three CLI subcommands shipped in five days and for the first time a complete simulation lifecycle — start, wait, cost, stop — is scriptable end-to-end; a returning external contributor fixed three separate reliability failures that thinking models and slow local LLMs had been exposing.

## What Shipped

### CLI gains start-to-stop automation

Before this week a script automating MiroShark had a structural gap: there was no primitive to block until a simulation finished, no terminal way to check what it cost, and no way to cancel a stuck run. Every integrator hand-rolled a poll loop against the API or left the sim running.

Three subcommands shipped in five days and closed all three gaps.

`wait` (#215, `959aef8`) polls `GET /api/simulation/<id>/run-status` on a monotonic deadline, exits 0 on `completed`, 1 on `failed`/`stopped`, and 2 on timeout; progress lines print to stderr so `--json` piping stays clean. `cost` (#208, `cef787b`) calls the existing `/cost.json` endpoint and prints `~$X.XXXX` with a per-phase breakdown (`graph_build`, `simulation`, `report`) and a `~` prefix that signals lower-bound — mirrors the embed pill's caveat. `stop` (#216, `ed3cfd0`) POSTs to `/api/simulation/stop`, prints `<sim_id> <runner_status>` on success, and exits 0/1.

`python cli.py wait "$SIM" && python cli.py cost "$SIM"` now works. Every integrator's hand-rolled poll loop is dead code. The "$1 to simulate anything" claim is checkable from a terminal, not just the embed widget.

### Dan-and hardens the engine for thinking models and slow LLMs

Daniel Andersen (dan-and) shipped three PRs in the same merge window targeting a coherent failure class: MiroShark running on thinking models or slow local LLMs, where prior assumptions about response shape and timing broke silently.

`fix(llm)` (#209, `7a9dffa`) is a 9-file hardening pass across the full inference pipeline. Thinking models (DeepSeek, Qwen3, MiniMax M2.5) produce `<think>` blocks that eat the `max_tokens` budget, leaving an empty response that callers' `json.loads()` crashes on. `llm_client.py` now pads `max_tokens` by a configurable `THINKING_BUDGET_TOKENS`, strips unclosed `<think>` blocks, and returns `None` on empty post-strip content so retry logic fires instead of crashing. `json_repair.py` gets a new pass stripping invalid escape sequences — Windows paths like `C:\Users` and LaTeX `\f` — that were breaking JSON parsing after think-block stripping. Six call sites across `graph_tools.py`, `simulation_config_generator.py`, `wonderwall_profile_generator.py`, `contradiction_detector.py`, and `ner_extractor.py` get `repair_truncated=True`. (+97/−21, 9 files)

`fix(interview)` (#214, `8bbf206`) addressed a different failure mode: Persona-interview dispatch could hang silently when a slow model didn't respond, showing a spinner in the UI with no error surfaced. The fix wraps `platform.running()` action dispatch in try/except that sends `None` back on error instead of blocking; `Step5Interaction.vue` now surfaces the failure state and error messages instead of spinning indefinitely. 10 files changed — `simulation_runner.py`, `platform.py`, `simulation_ipc.py`, `run_parallel_simulation.py`, `env.py`, `agent.py`, `api/simulation.py`, and the Vue UI — at +209/−79.

`fix(i18n)` (#213, `cf0b980`) fixed non-English locale drifting back to English mid-run. The one-time system-prompt locale directive was getting diluted by English environment observations accumulated over rounds; the fix appends a locale-aware language directive to every per-round user message in `perform_action_by_llm()` so the instruction persists through the simulation.

All three target the same gap: the engine was built and tested against cloud API models with fast, clean JSON responses. Real-world deployments use reasoning models, Ollama, and local LLMs that don't share those assumptions. These PRs are the compatibility layer.

## Fixes & Polish

- **Fan-out restored after #209 regression** — `#209`'s total-failure path in `_generate_sub_queries` collapsed PanoramaSearch from 4 semantic facets to a single bare query; `#211` (`9097055`) restored the 4-way default (`query`, `main participants`, `causes and impacts`, `development process`) so searches still fan out under LLM outage
