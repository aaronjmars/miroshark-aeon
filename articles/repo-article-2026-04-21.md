# MiroShark Grew a Brain: The Weekend a Simulator Became a Knowledge Graph

Most swarm simulators treat memory as a scratchpad — a transcript of what agents said, dumped into a vector index, retrieved by cosine similarity, forgotten when the run ends. MiroShark just stopped doing that. Over the weekend of April 20–21, one direct push and one pull request added a production-grade knowledge graph, a Model Context Protocol server, 14 features siphoned from four sibling forks, and the project's first standing test suite. If last week was about *analyzing* simulations, this week is about the substrate underneath them.

## Current State

At 750 stars and 146 forks as of Monday afternoon, MiroShark sits roughly 200 stars short of its April 30 self-imposed 1K target. The repo closed its ninth straight day with zero open issues and zero open PRs — a clean-slate cadence that has become the tell for when a big drop is about to land. On Sunday night it landed. Commit `b20f955` went in directly to main at 22:50 UTC: 17 files, +2,690/-93, a single message explicitly citing Hindsight, Graphiti, Letta, and HippoRAG as influences. Seventeen hours later, PR #41 merged: 43 files, +4,005/-29, sixty-two unit tests, and a GitHub Actions workflow to run them on every push. The hours that followed were a cleanup pass — breaking circular dependencies, strengthening weak type annotations, stripping AI slop and legacy fallbacks. Infrastructure, then housekeeping.

## What's Been Shipping

The graph memory stack is the structural move. MiroShark now writes every agent activity — beliefs, observations, facts — as a labeled edge in a Neo4j graph with four timestamps on it: `valid_at`, `invalid_at`, `created_at`, `expired_at`. Bi-temporality means you can query the graph *as of* any point in the past without destroying history. When two edges contradict, an LLM adjudicates and the loser gets invalidated rather than deleted. A Leiden clustering pass builds `:Community` subgraphs with their own LLM-written summaries and their own vector index, which a new `browse_clusters` tool lets the report agent zoom out to before zooming in. On retrieval, the pipeline does hybrid search (vector + BM25 + graph BFS from seed entities) and reranks with BGE-reranker-v2-m3. The ReACT reasoning trace — every thought, every tool call, every observation — is persisted as a traversable subgraph under `:Report → :ReportSection → :ReasoningStep`. You can interrogate why the report said what it said, one step at a time.

Then there's `backend/mcp_server.py`. Eight tools — `list_graphs`, `search_graph`, `browse_clusters`, `search_communities`, `get_community`, `list_reports`, `get_reasoning_trace`, and report listings — exposed over stdio to Claude Desktop. Point a local Claude at MiroShark's graph and ask questions of it directly, without opening the app.

PR #41 is the second half of the weekend. Four sibling forks — MiroJiang, MiroWhale, OpenMiro, and mirofish-oracle-seeds — had been experimenting in parallel. Their best ideas came home in one bundle, all opt-in behind env flags. Just Ask mode (`POST /api/simulation/ask`) skips the document requirement entirely: an LLM synthesizes the briefing from a question alone. Counterfactual branching forks a running simulation at a future round with a narrative injection, turning director mode into a tree of alternate histories. Per-round frame snapshots enable scrubbing UIs over long runs. `analyze_equilibrium` — a new report-agent tool — fits a 2-player stance game on the final belief distribution and enumerates Nash equilibria with `nashpy`. FeedOracle MCP seeds pipe live MiCA/DORA/macro data into the scenario doc at template load. Per-agent MCP tools let personas with `tools_enabled: true` parse `<mcp_call/>` tags mid-simulation and dispatch through pooled stdio subprocesses. Embedding batch size went from 32 to 128; Anthropic prompt caching slots in as a silent no-op on non-Claude models; a `is_public` flag plus `POST /api/simulation/<id>/publish` turns embed endpoints into 403s until explicitly released. A dependency-light `miroshark-cli` ships the whole API to the terminal.

## Why It Matters

The research motivation is explicit: bi-temporal graph memory with LLM adjudication is the design pattern popularized by Zep's Graphiti (June 2024) and extended by HippoRAG's neurobiologically-inspired retrieval. MCP as the inter-agent surface is Anthropic's protocol (November 2024) being treated as infrastructure rather than demo. Letta's persistent agent memory is the conceptual parent of the ReACT-trace-as-subgraph move. What's novel is the composition: a simulation engine whose every agent utterance is a temporally-scoped graph edge, whose every report is a traversable reasoning tree, and whose knowledge surface is MCP-addressable from outside the app. The Nash equilibrium tool is the academic tell — this is no longer just a demo platform. It's trying to become a research instrument.

The cleanup pass that followed — seventeen consecutive refactor and chore commits between 15:26 and 15:44 UTC — is worth noticing on its own. "Remove AI slop and stale comments," "remove defensive error-swallowing," "break circular dependencies," "strengthen weak type annotations." The kind of commits a codebase writes when it decides it wants to outlast this release cycle.
