### Theme 1: Eighth surface — Gallery Search & Filtering (PR #69, filed and merged today)

**Summary:** `/explore` was a reverse-chronological scroll over public sims; PR #69 turns it into a queryable research corpus. `GET /api/simulation/public` now accepts six new optional query params that compose with logical AND — `q` (case-insensitive substring on the scenario, ≤200 chars), `consensus` (`bullish` / `neutral` / `bearish` — same `±0.2` stance threshold every other surface uses), `quality` (`excellent` / `good` / `fair` / `poor` — first-word match on `quality_health`), `outcome` (`correct` / `incorrect` / `partial` — implies `verified=1`), `sort` (`date` default / `rounds` / `agents`), `page` (1-based alternative to `offset`). Empty / unknown values are no-ops on the server side, and the response envelope echoes every active filter while `total` reflects the *filtered* count (Stripe `total_count` convention) so the load-more "X remaining" hint and `has_more` flag stay accurate inside the active filter set. ExploreView ships a debounced 300ms search bar, two chip groups (Consensus + Quality), a sort dropdown, and a Reset button — all filter state mirrors URL params via `router.replace` so `/explore?q=aave&consensus=bearish&quality=excellent&sort=rounds` is bookmarkable and shareable.
