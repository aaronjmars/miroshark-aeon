# The Day MiroShark Watched Itself Spend Money: Four PRs, Three Hours, and a Default That Finally Costs $1

The repo's tagline since April 22 has been "Simulate anything, for $1 & less than 10 min." Today is the day that line started to be true by default instead of true if you knew what to set. Four PRs landed on `main` between 12:56 and 16:33 UTC — one shipping motion, one threading observability through every LLM call, one fixing the leaks the new traces immediately exposed, and one capping the default round count from 128 to 30–40. Read together, today is MiroShark turning the metering on, then turning the bill down.

## What's Been Shipping

Today's commits, in order:

**12:56 UTC — PR #50: Animated Belief Replay GIF.** A new endpoint `GET /api/simulation/<id>/replay.gif` that renders a 1200×630 GIF — one frame per simulation round, bullish/neutral/bearish bars sliding to each round's distribution, 60-frame cap with even subsampling that always keeps the final round, on-disk cache by content hash, the same publish gate as the share-card. The Embed dialog gets a tap-to-play preview that mounts the `<img>` only on click so opening the dialog doesn't pull GIF bytes for every viewer. Pure Pillow, zero new dependencies. Discord and Slack auto-play GIFs from a direct URL, so every share now ships motion as well as the still card from PR #42.

**14:09:24 UTC — PR #51: Langfuse trace metadata.** Every OpenRouter call from the simulator now lands in Langfuse with a `sessionId` (the sim ID), a `name` describing what kind of call it is — `report_outline`, `ner_extraction`, `agent_action`, eleven more — and tags rolling up to four phases (setup / round / report / ingest). The piece worth flagging is `TraceContext.wrap_fn(fn)`, a helper that snapshots and restores trace context across `ThreadPoolExecutor` workers, because Python's `threading.local` doesn't propagate across threads and reused workers carry stale state if you don't explicitly clear and set it. Once that landed, the question "where is the cost coming from?" stopped being a code-archaeology problem and started being a Langfuse filter.

**14:10:10 UTC — PR #52: Stop tool retry loop and bound research-call context.** Forty-seven seconds after #51, three fixes derived from what the new traces had just made visible. (1) Twelve idempotent social-platform actions — like, dislike, repost, follow, mute, several more — used to return `{success: False, error: "X record already exists"}` when they were already in the desired state, so agents retried the same tool four times per round and ballooned conversation history past 40K input tokens. They now return `{success: True, noop: True}`. (2) `max_iteration` was being stored on `self` but never plumbed to CAMEL's parent constructor, so the per-step ReAct loop was unbounded; default rises 1 → 3 with `prune_tool_calls_from_memory=True`. (3) `web_enrichment.py` caps `simulation_requirement` at 1,500 chars in entity-research prompts — a multi-KB user briefing was being pasted into every research call, 60–80K input tokens per entity.

**16:33 UTC — PR #53: Templates UX + capped default rounds + NoneType guards.** The cheapest line of code today: default round count clamped to `[30, 40]` instead of 128. For a typical 96-hour × 45-minute configuration, that's a 3-4× cut on the dominant line item of every default-config run. Custom mode still tracks the natural max for power users. Same PR adds NoneType guards on five social-platform handlers that subscripted `_get_post_type(post_id)['type']` without nil-checking, ships `polymarket` overlays on 5/6 templates instead of 1/6, retries Templates and Trending panels with 0/750/1500/3000ms backoff so they paint when Vite serves before Flask is warm, and makes history-modal file rows clickable through a path-traversal-guarded download endpoint.

## Why Today Matters

Today is the operational counterpart to the week's surfacing arc. The April 21–27 stretch shipped seven contracts — graph memory, share card, gallery, MCP, OpenAPI, webhook, `/verified` — that made MiroShark addressable from every direction a serious user might approach it. The thing that arc didn't answer is the question every reader of the README has been quietly asking since April 22: *does it actually cost a dollar?* PR #50 widens the distribution surface one more notch. PR #51 instruments the engine. PR #52 closes three specific leaks the instrumentation immediately surfaced. PR #53 touches the single biggest cost knob — the round count default — and turns it down by 3-4×. The "$1 & under 10 min" line on the GitHub description was a marketing claim on Wednesday. After today's four merges, it's closer to a default configuration. With two days left in the 1K-stars sprint, the timing is on the nose.
