# Sheet Music Outlasts the Performance

The performance ends when the last note dies. The recording captures one rendering — one orchestra, one room, one night. Sheet music outlasts both. A score is a *generator*: hand it to any competent ensemble in any century and they'll produce a performance that, by the criteria the score actually cares about, counts as the same piece. The recording is extensional. The score is intensional. One is a frozen output. The other is a recipe that keeps running.

This distinction is older than software — Frege made it the spine of modern logic, mathematicians use it to separate "two algorithms computing the same function" from "the same algorithm," and the [Stanford Encyclopedia of Philosophy](https://plato.stanford.edu/entries/logic-intensional/) treats it as foundational. In 2026 it has quietly become the most useful distinction for anyone building software whose outputs can be cheaply re-generated by a model.

## The output crisis is over because outputs are now cheap

The story of the last two years is that almost any output is now reproducible-shaped without being reproducible. Pictures, prose, predictions, simulations — a model will hand you another one in five seconds. What it won't hand you is the same one. The [reproducibility-as-spec literature](https://arxiv.org/pdf/1504.01310) calls this an extensional crisis: artifacts proliferate while the conditions that generated them stay implicit.

The response, across very different communities, has been to start treating *inputs* as the primary artifact. The [W3C PROV-AGENT specification](https://arxiv.org/pdf/2508.02866) extends provenance modeling for agentic workflows so that every model call carries its prompt, model version, tool configuration, and parent calls forward. The [Reproducible Builds project's 2026 report](https://dasroot.net/posts/2026/05/reproducible-builds-cpp-python-bazel-docker/) puts Fedora 45 at 90% bit-for-bit reproducibility, aiming for 99% by year-end. [NixOS Flakes](https://www.programming-helper.com/tech/nixos-2026-declarative-linux-reproducible-systems-rollbacks) is now the default deployment substrate for several enterprises, with a reported 90% drop in configuration drift. IP lawyers, of all people, have arrived at the same place: a wave of 2026 articles argues that [prompts, not outputs, are now the protectable trade secret](https://www.klemchuk.com/ideate/ai-prompts-new-form-of-intellectual-property) — copyright won't reach the AI output, but the prompt that produced it can be guarded like a recipe.

The pattern, stated cleanly: in a world where outputs can be regenerated, the value migrates to whatever lets you regenerate them.

## A small instance of the same move

MiroShark is an open-source platform that runs agent-based simulations — debates among synthetic personas reacting to a question, producing a consensus signal. Over the past two months its publicly published simulations have grown twenty-five "surfaces": ways to read a finished simulation. A direction signal at `/signal.json`. A peak-round inflection at `/peak-round`. A turbulence index at `/volatility`. Per-agent belief trajectories. A consensus badge SVG. An academic citation in BibTeX. A Polymarket-shaped JSON. An oEmbed provider. All twenty-five return *outputs* — the artifacts of a simulation that already ran.

On 2026-05-31, [PR #131](https://github.com/aaronjmars/MiroShark/pull/131) opened a twenty-sixth surface: `GET /api/simulation/<id>/clone.json`. It is the first surface that returns *inputs*. The body it serves is the exact JSON that `POST /api/simulation/create` accepts — same field set, same defaults, the same `polymarket_market_count: [1,5]` clamp, the same country-string normalization. The PR description states the move plainly: every other surface tells you what the simulation produced; this one tells you what produced it.

The change is small — 1,205 lines, 24 offline tests, no new dependencies. The category shift is not small. Twenty-five extensions of one simulation, then one intension. The first artifact the system serves that another deployment could replay.

## Why "wire-compatible" is the load-bearing detail

If `clone.json` returned a pretty-printed summary of the sim's parameters, this would be documentation. What it returns is a body that another instance of the platform — or a fork, or a third-party tool, or a future version of the same instance with the simulator rebuilt — will accept verbatim. The structural compatibility between the surface and the create endpoint is what gives the intension a chance of replaying as the same extension. Sheet music without orchestras to read it isn't a generator either; it's just notation.

The detail that earns its place: the cache header is `max-age=3600`, an hour, rather than the five minutes used by the analytical surfaces nearby. The reasoning, from the design notes, is that the analytical surfaces represent a simulation's state mid-evolution while clone.json represents its structural specification, which doesn't shift round-to-round. Inputs and outputs decay on different clocks. Acknowledging that in a single cache header is the same move as a librarian shelving scores and recordings in different rooms.

## What changes when systems surface their inputs

The platforms that survive this decade are quietly converging on a stance: the durable artifact is the one you can run again. Containers froze the environment. Nix and Bazel froze the build. PROV-AGENT freezes the workflow. Reproducible Builds freezes the bits. Prompt-as-IP doctrine freezes the instruction. In every case the same inversion: the specification is being lifted out of the output and given a URL of its own.

For agent-based systems — where outputs are stochastic, models change, and any given run is one draw from a distribution — this inversion matters more, not less. A simulation result is interesting; a simulation result that an external integrator can re-run against their own model, their own day's news, their own demographic filters is a different kind of object. It is a piece of sheet music for a question nobody can finish answering.

Twenty-six surfaces in, one of them finally hands the reader what would let them perform the piece themselves. The rest were the recording.
