## What Shipped

### Attestation — verifiable AI, for real

The biggest aeon ship this window: every skill run now gets a cryptographic proof it actually happened. Sigstore provenance, signed alongside the output, publicly logged to GitHub. Not "trust us it ran." Verify it.

PRs: #650 (workflow), #651 (docs), #652 (sign run manifest), #653 (immutable run-scoped snapshot).

Aaron announced it July 7: "attest anything on @aeonframework ⭐ — once a run is provably real, you can build on it: pay agents for verified work, gate on-chain actions behind proof-of-execution, let fleets trust each other without a human in the loop." 43 likes, 6 replies. This is the trust layer that makes autonomous agents composable.

### Secrets architecture overhaul (aeon)

The entire "sandbox blocks the network" paradigm is dead. 7 PRs in one day retired the prefetch model — scripts that ran before Claude started to cache API responses — and replaced it with in-run secretcurl calls with per-skill least-privilege secrets. XAI, CoinGecko, Alchemy, Etherscan, bd-radar, fleet, okf — all now live auth'd calls inside the skill run, not cached artifacts.

Key PRs: #684 (shrink static env, per-skill key injection), #685 (remove prefetch-xai.sh), #687 (route XAI through secretcurl), #689 (CoinGecko/Alchemy/Etherscan), #691 (bd-radar / fleet / okf), #695 (irreversible sends in-run). Also: docs updated to kill the myth that the sandbox blocks outbound network (#692).

This changes the architecture: no more pre-caching scaffolding, no more postprocess scripts for API calls. Skills call APIs directly, auth'd, inside the run.
