# Week in Review: Cost Visible, Languages Fixed, Self-Hosting Extended

*2026-06-22 ‚Äî Weekly shipping update*

> This week the "$1 to simulate" claim got a face: every public embed now shows the actual cost of the run via a new queryable endpoint ‚Äî and the locale-in-threads bug that was silently producing English output in DE/FR/ZH sessions closed after four hand-patches across as many call sites.

## What Shipped

### The "~$1" cost estimate landed where strangers actually see it

Two commits closed this gap in sequence. `a52ea34` (#179) introduced `GET /api/simulation/<id>/cost.json` ‚Äî a new `backend/app/services/cost_service.py` (+699/‚àí25 lines) that returns `estimated_cost_usd`, token totals, and per-model/per-phase breakdowns for any completed run. That made the dollar-figure real and checkable, but only for developers who know to call the endpoint.

`09a60cf` (#190) put it in the room where strangers are: the public embed widget now fetches `cost.json` on load and renders a "~$0.87" pill in the header meta row alongside status, round count, and agent count (`frontend/src/views/EmbedView.vue`, +68/‚àí1). The "$1 to simulate anything" claim now reads off the page next to every shared simulation result, not buried in an API response no one is looking at.

### Simulations in German, French, and Chinese stopped reverting to English mid-run

The week's heaviest workstream closed a chain of related bugs. The root class: Python's `ContextVar` (which carries the active locale) does not propagate into `ThreadPoolExecutor` workers. Both the report-agent (`3e054f4`, #194, by dan-and) and the `graph_tools` fallback interview (`165118d`, #198) were silently producing English output during multi-threaded sections of a DE/FR/ZH run. Both are now fixed with the same pattern: capture `get_active_locale()` on the parent thread, re-apply with `use_locale()` inside each worker.

Alongside the thread fix, the full French prompt locale shipped ‚Äî `5643802` (#186) translated all seven prompt modules in `backend/app/prompts/locales/fr/` from English stubs into working French copy (+556/‚àí42 across `graph_tools`, `social_simulations`, `ontology`, and four others). And `696f9ad` (#189, by dan-and, +2,888/‚àí1,973) added German to both the frontend UI and the agent profiles and communications ‚Äî not just the labels but the role-play text agents use mid-simulation ‚Äî while also correcting language-drift across ZH/FR/EN sessions that were sometimes switching languages mid-run. Two JSON-salvage fixes (`a9366b2` #188, `97e83fc` #192) closed the upstream cause: non-English and local-LLM responses were hitting a 700-token cap that truncated the JSON, crashing the suggestions step; the token limit is now 1,500 and `json_repair` salvages partial arrays rather than erroring.

### Self-hosters with local LLMs got a web-search path

`8bdaa73` (#178, by dan-and, +1,167/‚àí55) wired SearXNG and Firecrawl in as drop-in alternatives to the default web-enrichment stack. MiroShark's director-mode agents use internet access to enrich simulation context; previously that required an OpenAI-compatible setup with web-search tooling. Self-hosters running Ollama or any non-web-search LLM can now point `SEARXNG_BASE_URL` and `FIRECRAWL_API_URL` at their own infrastructure and the engine routes through them. The commit added a new `backend/app/utils/searxng_client.py` (+193 lines), two new Docker Compose files (`docker-compose.ollama.yml`, `docker-compose.traefik.yml`), four new test files, and a settings panel in the frontend where operators toggle the provider.
