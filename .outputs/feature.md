*Feature Built — 2026-04-18*

Agent Demographic Breakdown Panel
MiroShark now shows a Demographics tab next to Influence, Drift, and Network. It slices every simulation's agent pool by age range, gender, country, actor type (real people vs institutional accounts), and primary platform, then shows how each subgroup actually behaved — what their average final stance was, how much their opinions swung from round 1 to the final round, and how influential they became. The panel surfaces the most striking cross-group difference as a single KEY SUBGROUP DYNAMIC headline so the finding is visible without reading every row.

Why this matters:
The influence leaderboard tells you *who* was loudest; the belief drift chart tells you *when* the crowd shifted. Neither answered the subgroup question researchers keep asking — "did institutional accounts land in a different place from individuals?", "did Twitter-primary agents flip earlier than Reddit ones?", "was one age cohort more volatile?". Persona data already carried every demographic field needed (age, gender, country, source entity type), it just had no analytical lens. This closes that gap without any new data collection and complements Quality Diagnostics (PR #32) and Interaction Network (PR #33) that shipped yesterday, rounding out MiroShark's post-completion analytics suite. Picks up idea #4 from the Apr 15 repo-actions batch because the Apr 16 batch had only Multi-Document Mode left, which was explicitly deferred earlier today as a larger standalone PR.

What was built:
- backend/app/api/simulation.py: new GET /<sim_id>/demographics endpoint + five helpers. Loads reddit_profiles.json (falls back to twitter_profiles.csv), classifies each agent's source_entity_type as individual vs institutional using the OasisProfileGenerator taxonomy, buckets age into <18 / 18-24 / 25-34 / 35-44 / 45-54 / 55+, and joins against the first and final belief-position snapshots in trajectory.json so stance volatility is the actual round-0 → final delta. Caches the result in demographics.json; ?refresh=1 forces a recompute. Top-divergence helper sweeps all five dimensions and returns the largest cross-segment stance gap as a plain-English headline.
- frontend/src/components/DemographicBreakdown.vue: new overlay (~500 lines). Tab bar across the five dimensions with per-tab segment counts, one card per segment showing label + agent count, a stacked bullish/neutral/bearish distribution bar, and four metric columns (mean / σ / volatility / influence). KEY SUBGROUP DYNAMIC callout for the top divergence; muted variant when a run has no belief trajectory.
- frontend/src/components/Step3Simulation.vue: registers the component, adds a ◇ Demographics toggle alongside Influence / Drift / Network / Director with matching mutual-exclusion behavior, hides the timeline feed when the panel is visible.
- frontend/src/api/simulation.js: getDemographicBreakdown(simulationId, { refresh }) helper.

How it works:
The endpoint walks every agent in the persona file, extracts five demographic dimensions, and then joins by user_id into the trajectory snapshots to recover each agent's initial and final stance (averaged across all tracked topics). Per-segment aggregates are computed server-side as plain Python — no pandas, no NumPy — so the endpoint works in the same lightweight backend environment as Quality Diagnostics and Interaction Network. The classifier mirrors the OasisProfileGenerator's INDIVIDUAL_ENTITY_TYPES / GROUP_ENTITY_TYPES / INDIVIDUAL_TYPE_KEYWORDS lists so a "ceo" or "founder" maps to individual and "mediaoutlet" maps to institutional. Top divergence applies a ≥ 0.05 stance-gap and ≥ 2 agents-per-segment threshold to avoid noisy pairs on tiny pools. The frontend stacks the three stance percentages into a single horizontal bar so a "73% bullish / 18% neutral / 9% bearish" segment reads at a glance.

What's next:
Wire the top_divergence headline into the article generator prompt so generated articles cite the strongest subgroup finding automatically, and add a CSV export of the full demographics table for researchers who want to drop subgroup stats into a paper.
