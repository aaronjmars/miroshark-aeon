*Feature Built — 2026-05-02*

Live Spectator Watch Page
MiroShark just shipped a "watch live" page at `/watch/<simulation_id>`. Tweet that URL while a simulation is running and the link unfurls as a 1200×630 image card on Twitter / X / Discord / Slack — clicking through opens a minimal full-viewport broadcast view with a belief bar, round counter, and progress bar that update every 15 seconds in real time. When the simulation ends, the page transitions to a final result card with "View full simulation →" + "Fork this scenario →" CTAs.

Why this matters:
The past distribution arc landed six share surfaces (gallery card, share card PNG, replay GIF, Markdown + JSON transcript, RSS / Atom feed, trajectory CSV / JSONL — yesterday's PR #66) but every one of them serializes a *finished* simulation. The "tweet a sim that's running right now" format had no product home — the runner URL today opens the full SPA simulation view with chrome unrelated to spectating and no auto-unfurl as live state. `/watch/<id>` is the live-broadcast complement: same URL keeps a snapshot-perfect OG card after the run finishes, and the page itself transitions from live polling → final result + CTAs at the end. From repo-actions Apr 30 idea #5 — the cleanest unbuilt small-effort idea on the list once trajectory CSV / JSONL shipped yesterday.

How it works:
The watch page is a fully self-contained server-rendered HTML document — no SPA build dependency, no framework runtime. The Flask handler reads the on-disk `sim_dir/` (same files the embed-summary endpoint reads) and renders OG / Twitter tags + an inline `<style>` + an SSR initial-frame body + a JSON bootstrap blob in `<script type="application/json">` + a vanilla-JS poller in a final `<script>` tag. The poller hits `/api/simulation/<id>/embed-summary` and `/api/simulation/<id>/run-status` every 15 seconds (matching the existing SPA polling cadence), updates the DOM in place via the same updaters the SSR initial frame uses, backs off to 60 s on transient failures, gives up after a 6 h absolute timeout, and fires one trailing 4 s refresh once the runner status flips terminal so the final belief snapshot lands. The bootstrap blob exposes the ±0.2 stance threshold as a constant so a spectator who sees the share card on Twitter and clicks through to `/watch/<id>` doesn't see the numbers shift mid-flow. Works behind a restrictive CSP that allows only `img-src 'self'`, and even with JS disabled (the SSR HTML still shows a meaningful frame).
