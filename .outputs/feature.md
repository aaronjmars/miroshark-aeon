*Feature Built — 2026-06-01*

Private Share Link
MiroShark simulations now have a middle ground between fully public and fully private. An operator can mint a token-gated `/preview/<token>` URL, hand it to one specific person (a co-author, a stakeholder, an integrator running a debug share), and that recipient sees the full simulation results — without flipping the public flag, surfacing the sim on the gallery, or letting Google index the link.

Why this matters:
The sharing model has been binary since day one. Every sim was either `is_public=true` (anyone with the URL plus every crawler can read it, and `/explore` indexes it) or private (only the operator with the admin token can see it). There was no middle path. A researcher sending a finished sim to a co-author for review, an operator giving a stakeholder a pre-publication preview, an integrator handing another team a debug share — none of those workflows fit the binary gate. This was repo-actions May-30 #1, re-eligible from May-22 #1, and the only privacy-relevant feature on any of the unbuilt idea lists. It moves MiroShark from a publish-or-hide tool to a platform with a real selective-share primitive.

What was built:
- backend/app/services/share_link_service.py: new service module (~510 LoC, pure stdlib). `generate_token` mints 32-character URL-safe tokens via `secrets.token_urlsafe(24)` — 192 bits of entropy. `resolve_token` walks the per-sim subdirectories and returns the owning sim_id for valid, non-revoked, non-expired tokens. `revoke_token` is idempotent. `list_tokens` returns active tokens newest-first. Default expiry 30 days, clamped to [1, 365]. Path-traversal guard rejects anything outside `[A-Za-z0-9_-]` before touching the filesystem.
- backend/app/api/simulation.py: three admin-gated endpoints alongside the existing `/publish` — `POST /share-link` (mint), `GET /share-links` (list active), `DELETE /share-link/<token>` (revoke). Reuses the existing `require_admin_token` decorator and `Authorization: Bearer $MIROSHARK_ADMIN_TOKEN` scheme.
- backend/app/api/share.py: new `/preview/<token>` public landing route. Renders a different HTML template from the public `/share/<sim_id>` page — with `<meta robots="noindex,nofollow">`, no Open Graph / Twitter / Farcaster Frame / oEmbed tags, `Cache-Control: no-store`, and a "Private Preview" badge. Unknown / revoked / expired tokens all return the same 404 body so a probe can't tell the cases apart.
- backend/tests/test_unit_share_link.py: 18 offline pytest tests covering generate, resolve, revoke, list, expiry arithmetic, clamp behaviour, path-traversal guard, two-tokens-same-sim, and the case where the sim directory is removed after mint.
- frontend/src/components/EmbedDialog.vue: new "🔗 Private share links" panel alongside the public toggle. Expiry dropdown (1 / 7 / 30 / 90 / 365 days), Generate button, per-token list with one-click Copy and Revoke, remaining-days countdown.

How it works:
A token is stored as `<sim_dir>/share-tokens/<token>.json` — co-located with the simulation's data directory so deleting a sim takes its tokens with it (no orphan flat index to clean up). The resolve path scans `os.listdir(sim_root)` and probes each subdir for the token file — O(N) over sim count with one `os.path.exists` call per sim, sub-millisecond on a warm filesystem for typical deployments. Atomic writes use the same staging-tempfile plus `os.replace` posture `surface_stats` uses, so a failed write never leaves a half-written record. The critical privacy boundary: the token grants the preview page only — `signal.json`, `share-card.png`, `chart.svg`, `transcript.md`, and every other per-sim REST surface keep the existing `is_public` gate. This isn't a 26th publish-gated surface; it's the platform's first **third-state** sharing primitive: public / token-gated / private.

What's next:
The clipboard copy could be extended with a QR code button for in-person handoffs, and the panel could surface a "Share with QR" path. A future surface would be `GET /api/share-link-stats` to count how many times each token has resolved, giving operators a delivered/seen signal without exposing the recipient. The natural follow-on is reusing the same token primitive for time-limited integrator API keys — `secrets.token_urlsafe(24)` + an `<sim_dir>/integration-tokens/<token>.json` parallel.
