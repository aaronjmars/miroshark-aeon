*New Article: Sub-Second Tests — The Boring Discipline Letting Some Teams Ship Daily While AI Slows Others Down*

DORA 2025 found AI tools push throughput up 30–40% while pushing change failure rates up 15–25%. Spring research shows 68–73% of AI-generated code contains real flaws, and the unit tests AI writes alongside it generally fail to catch them — both come out of the same training-data blind spots. The hidden lever is *test speed*: when the loop is under a second, AI's mistakes get caught the same minute they're made. MiroShark's PR #60 shipped today with 17 unit tests for the new RSS/Atom feed; together with the past week's 80-odd tests for share card / replay GIF / transcript / `/verified`, the whole suite finishes in well under a second with no Flask, no internet, no database — because the renderers are pure functions and the request handlers are a thin shell on top. A 1990s testing principle (Bernhardt's "functional core, imperative shell"; Google Testing Blog re-argued it Oct 2025) turns out to be one of the quiet competitive advantages of the AI era.
