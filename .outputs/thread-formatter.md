Topic: MiroShark contributor guide — PR #162

1/ MiroShark had 1,270 stars and no real contribution guide. PR #162 fixes that.

2/ The old CONTRIBUTING.md was one section: run the tests. How to set up the env, extend the API, open a PR — contributors reverse-engineered all of it from package.json and the CI config. a 1,200-star repo shouldn't work this way.

3/ PR #162 ships three sections: dev setup (setup:all, maps exactly to package.json), submitting a PR (branch → CI → review), and adding an API endpoint. English + zh-CN on merge.

4/ 269 forks. 3 confirmed integrators building on top. the constraint was never interest — it was knowing where to start. a real guide converts forks into PRs.

5/ the contributor guide is up — first PR welcome. https://github.com/aaronjmars/MiroShark/pull/162 🦈
